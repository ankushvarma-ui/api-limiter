What is an
API Rate Limiter?

A simple visual explanation 👇

🍋 Think of it like this
🏪

A shop with a entry rule

A shop allows 100 customers per hour. If someone tries to enter for the 101st time — the door locks. If they keep trying too many times — they get banned for 1 hour.

⚙️ How it works — step by step
🌐

Someone sends a request

A user, app, or bot tries to access your API (like logging in or getting data).

↓
🔍

Check: Is this IP blocked?

We look up their IP address. If they were blocked before — immediately rejected.

↓
🔢

Count their requests

We count how many requests this IP sent in the last 15 minutes. Over 100? Rejected.

↓
⚠️

Repeated violations = Auto-block

If they exceed the limit 3+ times, their IP is blocked for 1 full hour automatically.

↓
✅

Normal users pass through

Regular users never hit the limit. They get their data, everything works fine.

📬 What the user gets back
✅

Normal Request

Under 100 requests — gets their data back normally

⏳

Too Many (429)

"Too many requests. Try again in 15 minutes."

🚫

IP Blocked (403)

"You are blocked for 1 hour due to repeated abuse."

100

per 15 minutes

💳

Stripe

🗺️

Limits map requests per day per key

🛡️ In one sentence

A Rate Limiter is like a security guard for your API — it lets normal users in, slows down heavy users, and permanently blocks abusers. Every real-world API needs one.
